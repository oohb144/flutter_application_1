# K230 联机方案 — 电脑端开发接口说明

> K230 只做推流+音频播报+HTTP 命令，电脑端 Flutter+VLC 拉流+人脸识别。
> 本文档面向**电脑端 Flutter 开发者**，重点说明 K230 的 HTTP API（电脑端要调用的接口）。

## 一、架构

```
K230 端                          电脑端（Flutter，本目录开发）
─────────────────                ─────────────────────────────
摄像头采集                        Flutter app
  ↓ CHN0                         + VLC 拉流 rtsp://<K230_IP>:8554/test
VENC H264 → RTSP 推流 ──────→    + 人脸识别（Python 后端 InsightFace/Dlib，或 mlkit）
  ↓                              + GUI（识别结果/人脸库/状态显示）
HTTP server :8001                + HTTP 下发命令/识别结果
  GET /status  ←─────────────     POST /command  {rtsp/speak/led/exit}
  POST /command ←────────────     POST /face_result {label,known,score}
  POST /face_result ←──────────
  ↓
AudioPlayer 播报（TTS/wav）
LED + 按键 + OSD 状态
```

## 二、K230 端 HTTP API（电脑端调用）

**Base URL**: `http://<K230_IP>:8001`（K230 上电后连 WiFi，OSD 屏显 IP，如 `192.168.123.183`）

**CORS**: 已开启 `Access-Control-Allow-Origin: *`，Flutter Web 可直接跨域调用；App 端无跨域问题。

### 1. `GET /status` — 查询 K230 状态

响应 JSON：
```json
{
  "ip": "192.168.123.183",
  "rtsp_url": "rtsp://192.168.123.183:8554/test",
  "rtsp_running": true,
  "state": "推流中",
  "audio_busy": false
}
```

Flutter 用途：状态栏显示 K230 是否在线、推流是否开。

### 2. `POST /command` — 下发命令

请求体 JSON：
```json
{"cmd": "<命令名>", ...参数}
```

响应 JSON：
```json
{"ok": true, "msg": "queued"}
```

**命令清单**：

| cmd | 参数 | 说明 |
|-----|------|------|
| `rtsp` | `{"cmd":"rtsp","on":true}` / `"on":false` | 开/关 RTSP 推流 |
| `speak` | `{"cmd":"speak","text":"欢迎回来"}` | K230 离线 TTS 播报中文 |
| `play_wav` | `{"cmd":"play_wav","file":"/data/audio/xxx.wav"}` | 播放 K230 本地 wav（兜底） |
| `led` | `{"cmd":"led","color":[255,0,0]}` | LED 颜色 [r,g,b] |
| `exit` | `{"cmd":"exit"}` | 退出 K230 主程序 |

### 3. `POST /face_result` — 推送人脸识别结果

电脑端识别到人脸后，把结果推给 K230 触发播报。

请求体 JSON：
```json
{"label": "user_1", "known": true, "score": 0.92}
```
- `label`: 识别出的人脸标签（熟人标签名，陌生人可空或 "unknown"）
- `known`: true=熟人，false=陌生人
- `score`: 置信度（可选）

K230 收到后播报：
- 熟人 → TTS「欢迎回来 {label}」（具体播报词 K230 端定）
- 陌生人 → TTS「发现陌生人」

响应：`{"ok": true, "msg": "queued"}`

### 4. `OPTIONS`（CORS 预检）

Flutter Web 跨域时浏览器自动发 OPTIONS，K230 返回 204 + CORS 头。App 端不需要。

## 三、电脑端 Flutter 要做的

1. **拉流显示**：用 VLC Flutter 插件（如 `flutter_vlc_player`）拉 `rtsp://<K230_IP>:8554/test` 显示画面
2. **人脸识别**：
   - 方案A（推荐，精度高）：Python 后端（FastAPI + InsightFace），Flutter 截 RTSP 帧 HTTP 发后端识别
   - 方案B（轻量）：Flutter 用 `google_mlkit_face_detection`（只检测，不识别）
   - 方案C：电脑端 OpenCV 直接拉 RTSP 流做识别（不经过 Flutter），Flutter 只做 GUI
3. **GUI**：识别结果、人脸库管理、状态栏（GET /status 轮询）、命令按钮（POST /command）
4. **触发播报**：识别到人脸 → POST /face_result → K230 播报
5. **控制 K230**：按钮开/关推流（POST /command rtsp）、发播报（POST /command speak）

## 四、端到端流程

```
1. K230 上电 → 连 WiFi → 屏显 IP → 自动开 RTSP 推流 → HTTP server :8001 就绪
2. Flutter 启动 → 输入 K230 IP → VLC 拉流显示画面 + GET /status 显示状态
3. Flutter（或 Python 后端）从 RTSP 流截帧 → 人脸识别 → 得 label/known
4. Flutter POST /face_result {label,known} → K230 TTS 播报「欢迎 label」
5. Flutter 按钮可 POST /command {rtsp/speak/led} 控制 K230
```

## 六、验证里程碑

1. **M1 推流长跑**：K230 长跑 5 分钟不卡，VLC 拉流看到画面
2. **M2 HTTP**：`curl http://<IP>:8001/status` 返回 JSON；`curl -X POST /command -d '{"cmd":"rtsp","on":false}'` 开关推流
3. **M3 音频**：`curl -X POST /command -d '{"cmd":"speak","text":"欢迎回来"}'` K230 出声
4. **M4 face_result**：`curl -X POST /face_result -d '{"label":"user_1","known":true}'` K230 播报
5. **M5 联机**：Flutter+VLC+识别+POST face_result → K230 播报，端到端跑通

## 七、关键风险（K230 端，电脑端开发不受影响）

- K230 不跑 KPU 不取 CHN2，推流可能因 sensor 流转停卡（K230 端上板验证，若卡改用 WBCRtsp）
- `speech.TTS` 可能依赖麦克风初始化（不行用 wav 兜底）
- 播放阻塞主循环（独立线程，播报时推流短暂降帧可接受）

电脑端开发可独立进行，K230 端 HTTP API 稳定后即可对接。
