# 在线语音指令对照表

> 触摸屏侧边栏「语音」按钮 → 录 4 秒 → 上传 DashScope 识别 → 整句中文子串匹配下表关键词 → 命中即触发对应命令。
> 关键词来源：`config.ONLINE_VOICE_KEYWORDS`；命令派发见 `main.py` 的 `_dispatch_http_command`。

## 关键词 → 命令 → 功能

| 说出关键词 | 命令名 | 功能 | 当前实现 |
|-----------|--------|------|---------|
| 主界面 / 主页 | `home` | 切到待机态（IDLE，停止识别/录入） | ✅ 已实现 |
| 录入 | `enroll` | 切到录入态（ENROLLING，随后短按/触摸录入人脸） | ✅ 已实现 |
| 开始识别 | `recognize` | 切到识别态（RECOGNIZING，实时检测+识别画框） | ✅ 已实现 |
| 停止 | `stop` | 切到待机态（IDLE，与「主界面」同义） | ✅ 已实现 |
| 开始 | `start` | （预留）语义不明确，未派发具体动作 | ⚠️ 未实现（命中会打印「未知命令」） |
| 打开录制 | `auto_record_on` | （预留）自动录制开启 | ⚠️ 未实现（远程录制功能阶段 D-1 再做） |
| 关闭录制 | `auto_record_off` | （预留）自动录制关闭 | ⚠️ 未实现（同上） |

## 交互流程

1. 按下「语音」按钮：
   - 若 RTSP 正在推流 → 先暂停推流（麦克风与 VENC 互斥），录音结束后自动恢复；
   - 蜂鸣器短响一声，屏幕显示「请说话 4s...」并倒计时。
2. 说完一句关键词（4 秒内）。
3. 录音结束，麦克风释放，屏幕显示「[MIC] 识别中...」。
4. 服务端回发整句 → 子串匹配关键词：
   - 命中 → 执行上表命令，成功提示音；若该命令会切界面（如识别/录入/待机），则不再自动恢复推流，尊重语音意图；
   - 未命中 → 提示「未识别到关键词」，若之前暂停过推流则自动恢复推流。

## 备注

- 子串匹配：整句中包含关键词即触发，例如说「请回到主界面」会命中「主界面」→ `home`。
- 录音期间人脸识别仍在跑（KPU），仅麦克风短暂占用，不与 VENC 并发。
- `start` / `auto_record_on` / `auto_record_off` 三条命令已登记关键词但未接业务逻辑，需后续在 `_dispatch_http_command` 补分支。
