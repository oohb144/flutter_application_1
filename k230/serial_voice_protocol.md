# K230 串口语音模块通信协议

> K230 人脸识别智能系统 — 串口语音模块（外接离线语音模块）通信约定
> 对应代码：`serial_voice.py` ｜ 集成：`main.py`

## 一、概述

K230 资源不足以同时跑人脸识别 + 本机音频采集（单 GIL + KPU 唯一 + 媒体管线唯一，
本机 KWS 麦克风采集还有问题）。故采用 **职责外移**：外接离线语音模块
（SU-03T / ASRPRO 等）做语音识别，K230 通过 UART 收命令码、发播报码，
**K230 完全不碰音频**，根治卡顿与采集问题（CLAUDE.md 阶段 C-2 方案）。

## 二、接线

K230 用 `ybUtils.YbUart`，默认 **UART1**，引脚：

| K230 引脚 | 方向 | 接 | 语音模块引脚 |
|-----------|------|----|--------------|
| Pin9 (TX) | 输出 | → | RX |
| Pin10 (RX) | 输入 | ← | TX |
| GND | — | 共地 | GND |

- 波特率 **115200**（`config.SERIAL_BAUDRATE`，模块端需一致）
- 电平：3.3V TTL。若模块为 5V，中间加电平转换或串电阻。

## 三、帧格式

```
┌────────┬──────┬────────┬────────┬──────────┬────────┐
│ 帧头   │ cmd  │ len    │ data   │ checksum │ 帧尾   │
│ AA 55  │ 1字节│ 1字节  │ N字节  │ 1字节    │ 55 AA  │
└────────┴──────┴────────┴────────┴──────────┴────────┘
```

- **帧头** `0xAA 0x55`，**帧尾** `0x55 0xAA`
- **cmd**：命令码（见下表）
- **len**：data 字段字节数（无 data 时为 0）
- **data**：可选载荷
- **checksum**：`cmd ^ len ^ data[0] ^ ... ^ data[n-1]`（按位异或，结果 `& 0xFF`）

### 校验和计算示例

发 `HOME`（cmd=0x01，无 data）：
```
checksum = 0x01 ^ 0x00 = 0x01
完整帧：AA 55 01 00 01 55 AA
```

发 `RECOGNIZE`（cmd=0x02）：
```
checksum = 0x02 ^ 0x00 = 0x02
完整帧：AA 55 02 00 02 55 AA
```

## 四、接收命令（语音模块 → K230）

语音模块识别到命令词后，按上表发帧给 K230。K230 在接收线程解析，入队，
主循环消费执行（线程安全）。

| cmd | 常量名 | 含义 | K230 动作 |
|-----|--------|------|-----------|
| 0x01 | HOME | 主页/待机 | → IDLE |
| 0x02 | RECOGNIZE | 开始识别 | → RECOGNIZING |
| 0x03 | ENROLL | 开始录入 | → ENROLLING |
| 0x04 | STOP | 停止 | → IDLE |
| 0x05 | RTSP_ON | 开推流 | 进识别态 + 开 RTSP |
| 0x06 | RTSP_OFF | 关推流 | 关 RTSP |
| 0x07 | ENROLL_CAPTURE | 录入当前帧 | 进录入态 + 录入 |

> 未识别的 cmd 会被打印 `未知(0xXX)` 并丢弃。

## 五、发送命令（K230 → 语音模块，播报）

K230 在状态切换 / 识别结果 / 录入结果时主动发给语音模块，模块端可配
收到码后播报对应语音（如"已进入识别模式""欢迎回来""录入成功"）。

| cmd | 常量名 | 触发时机 |
|-----|--------|----------|
| 0x10 | STATE_IDLE | 进入 IDLE |
| 0x11 | STATE_RECOGNIZING | 进入 RECOGNIZING |
| 0x12 | STATE_ENROLLING | 进入 ENROLLING |
| 0x20 | FACE_KNOWN | 识别到熟人（预留，识别逻辑接入后发） |
| 0x21 | FACE_UNKNOWN | 识别到陌生人（预留） |
| 0x30 | ENROLL_OK | 录入成功 |
| 0x31 | ENROLL_FAIL | 录入失败 |

> 状态播报 0x10/0x11/0x12 已在 `main.py` 的状态 enter 回调里自动发；
> 0x20/0x21/0x30/0x31 需在识别/录入逻辑里调用 `self._serial.send_face_result(...)` / `send_enroll_result(...)`。

## 六、语音模块端配置示例（SU-03T / ASRPRO）

在模块的图形化编程工具里，把识别词映射成"串口发送字节"：

| 识别词 | 发送字节（hex） |
|--------|-----------------|
| "主页" / "待机" | `AA 55 01 00 01 55 AA` |
| "开始识别" / "识别" | `AA 55 02 00 02 55 AA` |
| "录入" | `AA 55 03 00 03 55 AA` |
| "停止" | `AA 55 04 00 04 55 AA` |
| "开推流" / "推流" | `AA 55 05 00 05 55 AA` |
| "关推流" | `AA 55 06 00 06 55 AA` |
| "录入当前" / "拍一张" | `AA 55 07 00 07 55 AA` |

模块收到 K230 的播报码（0x10/0x11/0x20 等）时，可配"串口接收字节→播放语音"：
- 收 `AA 55 10 00 10 55 AA` → 播"已进入待机"
- 收 `AA 55 11 00 11 55 AA` → 播"开始识别"
- 收 `AA 55 20 00 20 55 AA` → 播"欢迎回来"
- ……

## 七、调试

- **独立测试** `serial_voice.py` 的 `__main__`：上电后每 5 秒发一次状态播报，
  收到命令打印 `cmd=0xXX data=...`。可用 USB-TTL 接 Pin9/10 自测回环。
- **main.py 集成后**：串口初始化成功打印 `[串口语音] 已启动`；
  收到命令打印 `[串口语音] 收到: RECOGNIZE data=b''` + `[主] 执行串口命令: 0x2`。
- 没接模块时 main.py 仍可跑（接收线程空转，按键兜底）。
